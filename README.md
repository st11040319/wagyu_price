# Git Lesson

## リモートリポジトリとローカルリポジトリとはそれぞれ何でしょうか？

リモートリポジトリはインターネット上に配置して複数人で共有するためのリポジトリ（GitHub）<br>ローカルリポジトリは開発者が使うために自分のPC上に配置するためのリポジトリ（自分のPC）

## プッシュとマージの違いは何でしょうか？

プッシュはローカルリポジトリからリモートリポジトリに反映させるためのコマンド<br>マージはローカルリポジトリの別ブランチの編集内容をmainブランチに取り込ませるためのコマンド

## コミットとプッシュの違い

コミットはステージングに追加されたファイルを記録するコマンド（セーブデータを保存する）<br>プッシュはコミットされた内容をローカルリポジトリからリモートリポジトリに反映させるコマンド

## コミットのメッセージはどのように書いてあげるのが最適でしょうか？

正確な編集内容を表すようなメッセージを表記してあげる

## ローカルでマージするフローと、プルリクエストでマージするフローの違いは何でしょうか？

ローカルでもマージ,フローは可能だが、自分自身で更新するためコードにバグがあっても気づかない可能性がある。<br>プルリクエストでマージ,フローの場合レビュー担当者に確認してもらえるため事前にバグを発見することができる

## コンフリクトを起こしてしまった場合、どう対処すべきですか？

* 先にマージされた変更内容を取り込む
* 後にマージされた変更内容を取り込む
* どちらの変更内容も取り込む
